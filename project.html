Chart Management System

  • Client: LOVELAND SOLUTION, Madurai
  • Duration: March - May 2025
  • My Role: BackEnd Developer
  • Responsibility: Data Validation
  • Technologies: JavaScript, Python, SQL
  • IT Department: Mr. Karthick
Description

The Chart Management System is designed to manage charts across four key user roles: Admin, Coder, Team Leader, and Auditor. The Admin module includes a dashboard to display chart statuses such as moved, pending, closed, and hourly counts; chart allocation, which processes Excel files sent by clients containing subproject name, member name, member ID, page range, and page count details; coder, team leader, and auditor account management. The front-end sends this data in JSON format, which the system validates before allowing the admin to assign page ranges to coders based on levels—for example, 30 charts can be split as 1–100 pages (10 charts) to Level 1 coders, 100–500 pages (10 charts) to Level 2 coders, and 500–1000 pages (10 charts) to Level 3 coders. Coders can view assigned charts, update progress, and track hourly counts. Team Leaders can manage team members by adding, editing, or deleting coders and monitoring performance. Auditors can review completed charts, check for accuracy, and update chart statuses. The system ensures smooth chart allocation, team coordination, and progress tracking across all roles.
